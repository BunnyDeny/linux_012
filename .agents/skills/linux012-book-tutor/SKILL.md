# linux012-book-tutor

## 工作流

当用户分享书中内容或提出相关问题时，按以下顺序执行：

### 1️⃣ 核心答疑（预热）

- 仔细阅读用户提供的内容，识别最容易混淆的概念
- 主动解释 3~5 个最核心的困惑点（如 `ldsw` 的真实行为、段寄存器变化、内存复用逻辑等）
- **先输出答疑，再邀请用户提出更具体的问题**
- 不要等用户列出一堆问题才开始回答 —— 要主动预判

### 2️⃣ 创建代码文件夹

- 参考工作区中已有的章节文件夹（如 `01_first_bootsect`、`02_bootsect_to_setup`）
- 创建新的 `0x_xxx` 文件夹，包含：
  - `bootsect.S`（如与已有章节相同可直接复用）
  - `setup.S` 或对应的核心代码文件
  - `Makefile`
  - `tools/build.c`（如需要镜像拼接）
- **代码必须能编译并通过 QEMU 运行**
- 运行验证：`make && qemu-system-i386 -fda linux.img`

### 3️⃣ 撰写详尽 README.md

README 是本章的知识沉淀，详细程度必须参考已有章节的 README（尤其是 `02_bootsect_to_setup/README.md`）。

#### 📋 README 内容要求

- 🌱 **本章引言**：说明本章在整体学习路径中的位置和核心目标
- 📁 **项目结构**：列出文件清单和各自职责
- 🔨 **编译 & 运行**：提供 `make` 和 `qemu` 命令，列出产物说明
- 📜 **代码逐行详解**：逐段解析关键代码，不能遗漏难点
  - 对每条指令说明输入/输出寄存器
  - 对隐含操作数指令（如 `lodsb`、`movsw`、`stosb`）特别说明段寄存器依赖
  - 用表格整理 BIOS 中断参数和返回值
- 🗺️ **内存布局**：展示各阶段内存使用情况
- 🎯 **启动流程图**：用箭头图展示完整启动时序
- 🧠 **关键概念回顾**：用表格做一句话总结

#### 🎨 README 风格要求

- **大量使用 emoji** 增强可读性和视觉层次
- 每个一级/二级标题前必须有 emoji（参考：`## 🌱`、`## 📁`、`## 🔨`、`## 📜`）
- 每个三级标题前必须有 emoji（参考：`### 🧭`、`### 📖`、`### 🔧`）
- 关键提示用 💡，警告/注意用 ⚠️，前后联动用 🔗，流程/阶段用 🎯/🔄
- 列表序号用 1️⃣ 2️⃣ 3️⃣ 等带圈数字 emoji
- 概念解释用表格呈现，表格标题行简洁明了

### 4️⃣ 等待深入提问

README 和代码输出后，主动询问用户是否有更具体的问题。
- 引导用户关注：指令寻址方式、段寄存器变化、BIOS 中断返回值、隐含操作数等方向
- 回答要精确到寄存器级别
