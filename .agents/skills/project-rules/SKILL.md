# 🎯 Project Rules

## 📝 Git Commit Language

Write all git commit messages in **Chinese** 🇨🇳.

- Commit message subject line must be in Chinese.
- Commit message body, if present, must also be in Chinese.
- Do not use English for commit summaries or descriptions.

## 😄 Markdown Emoji Style

When writing any markdown documentation (README, skill files, docs, etc.), use GitHub emojis generously to improve visual appeal and readability:

- Add section headers with relevant emojis (e.g., `## 🚀 Quick Start`, `## 📋 Requirements`).
- Use emojis in lists, notes, warnings, and tips.
- Use emojis to highlight important points or actions.
- Prefer expressive emojis that match the content context.
